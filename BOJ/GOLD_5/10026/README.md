## 문제

https://www.acmicpc.net/problem/10026

### 문제 파악하기

위 문제는 백준 알고리즘 10026 적록색약 문제이다.

해당 문제는 N x N인 그리드에서 같은 색으로 모여있는 구역의 수를 찾는 문제였다.

그리드를 탐색하는 방식에는 대표적으로 깊이 우선 탐색과 너비 우선 탐색이 있는데 해당 문제는 너비 우선 탐색으로 풀어야겠다는 생각을 했다.

<aside>
💡

인접한 칸만 확인하기 때문에 하나의 경로를 깊게 탐색하는 것이 아니라 면적을 넓게 탐색해야겠다는 결론을 내렸다.

</aside>

또한 색약인지 아닌지에 대한 정보를 bfs 함수의 파라미터로 넘겨 분기처리를 진행하기로 결정했다.

## 풀이

- bfsBoard 함수에서 가장 중요한 부분은 continue를 호출하는 부분인데 설명은 아래와 같다.

<aside>
💡

1. 내가 이동할 좌표가 실제로 도달할 수 없는 좌표라면 continue로 다음 탐색을 실시
2. 색약이 아니라면, 현재 좌표의 색깔과, 인접한 좌표의 색깔이 일치하지 않다면 탐색할 대상에서 제외한다. → 인접한 칸의 색이 다르다면 이미 다른 구역으로 구분되기 때문
3. 색약이라면, 빨강색과 초록색을 같은 색으로 보기 때문에 현재 색과 인접한 칸의 색이 다르면서도 둘 중에 하나라도 파란색이 포함되어 있다면 탐색할 대상에서 제외했다.
</aside>

- 또한 N \* N 모든 좌표에 대해 bfsBoard 함수를 호출하지만, 이미 방문한 적이 있는 좌표라면 bfsBoard 함수를 호출하지 않았다.

<aside>
💡

이미 탐색된 적이 있는 좌표이고, 이 좌표는 이미 인접된 칸들에 의해 같은 구역인지 판별되었기 때문 !

</aside>

- 마지막으로 bfsBoard가 수행될때마다 count 혹은 countBlind를 증가하여 총 몇개의 구역이 생기는지 판별한다.
