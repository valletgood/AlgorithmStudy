# 용액

[백준 2467번: 용액](https://www.acmicpc.net/problem/2467)

## 문제 파악하기

산성/알칼리성 용액들이 오름차순으로 정렬되어 주어질 때, **두 용액을 혼합하여 특성값이 0에 가장 가까운 조합**을 찾는 문제입니다.

### 조건

- 용액의 특성값: -1,000,000,000 ~ 1,000,000,000
- 같은 용액을 두 번 사용 불가
- 이미 오름차순 정렬되어 주어짐

> [!NOTE]
> 정렬된 배열에서 합이 특정 값(0)에 가장 가까운 두 수를 찾는 문제는 **투 포인터**로 O(N)에 해결할 수 있습니다!

## 풀이

**투 포인터** 알고리즘으로 해결합니다.

### 투 포인터 접근

```javascript
let l = 0;
let r = N - 1;
let diff = Infinity;
let answer = [];

while (r > 0 && l < N - 1) {
  const sum = cases[l] + cases[r];
  if (Math.abs(sum) < diff) {
    diff = Math.abs(sum);
    answer = [cases[l], cases[r]];
  }
  if (sum < 0) l++;
  else if (sum > 0) r--;
  else break;
}

console.log(answer);
```

### 동작 원리

1. **초기화**: `l`은 가장 왼쪽(최솟값), `r`은 가장 오른쪽(최댓값)
2. **합 계산**: 두 용액의 합 `sum = cases[l] + cases[r]`
3. **포인터 이동**:
   - `sum < 0`: 합을 키워야 함 → `l++` (더 큰 값 선택)
   - `sum > 0`: 합을 줄여야 함 → `r--` (더 작은 값 선택)
   - `sum === 0`: 최적해 발견 → 종료

> [!TIP] > **왜 투 포인터가 동작하는가?**
>
> 배열이 정렬되어 있으므로:
>
> - `l`을 증가시키면 합이 커짐
> - `r`을 감소시키면 합이 작아짐
>
> 이 성질을 이용해 0에 가까워지는 방향으로 포인터를 이동하면 최적해를 찾을 수 있습니다.

### 예시

```
용액: [-99, -2, -1, 4, 98]

l=0, r=4: -99 + 98 = -1  (diff=1, 저장)
l=1, r=4: -2 + 98 = 96   (sum > 0, r--)
l=1, r=3: -2 + 4 = 2     (sum > 0, r--)
l=1, r=2: -2 + (-1) = -3 (sum < 0, l++)
l=2, r=2: 종료

답: [-99, 98]
```

## 핵심 아이디어

1. **정렬된 배열**: 이미 정렬되어 있어 투 포인터 적용 가능
2. **양 끝에서 시작**: 최솟값과 최댓값부터 시작
3. **0에 수렴**: 합의 부호에 따라 포인터를 이동하여 0에 가까워짐

### 시간 복잡도

- 각 포인터가 한 방향으로만 이동
- **O(N)**
