# 오큰수

[백준 17298번: 오큰수](https://www.acmicpc.net/problem/17298)

## 문제 파악하기

수열의 각 원소에 대해 **오큰수(NGE)**를 구하는 문제입니다. 오큰수란 해당 원소의 오른쪽에 있으면서, 해당 원소보다 큰 수 중 **가장 왼쪽**에 있는 수입니다. 오큰수가 없으면 -1입니다.

> [!NOTE]
> 단순히 이중 반복문으로 풀면 O(N²)으로 시간 초과가 발생합니다.
>
> **스택**을 활용하면 각 원소가 스택에 한 번 push되고 한 번 pop되므로 **O(N)**에 해결할 수 있습니다.

## 풀이

이 문제는 **스택(Stack)**을 사용해 효율적으로 해결할 수 있습니다.

### 1. 초기화

```javascript
let answer = Array(N).fill(-1);
const stack = []; // 인덱스 저장
```

- `answer`: 오큰수 결과 배열 (기본값 -1)
- `stack`: 아직 오큰수를 찾지 못한 원소들의 **인덱스**를 저장

### 2. 스택을 활용한 탐색

```javascript
for (let i = 0; i < N; i++) {
  while (stack.length && nums[stack[stack.length - 1]] < nums[i]) {
    answer[stack.pop()] = nums[i];
  }
  stack.push(i);
}
```

> [!TIP]
> 스택에는 **값**이 아닌 **인덱스**를 저장하는 것이 핵심!
>
> - 현재 원소 `nums[i]`가 스택 top에 있는 인덱스의 원소보다 크다면
> - → 스택 top 인덱스의 오큰수는 `nums[i]`
> - → pop하고 answer에 기록
> - 더 이상 작은 원소가 없을 때까지 반복 후, 현재 인덱스를 push

### 동작 예시

수열 `[3, 5, 2, 7]`의 경우:

| i   | nums[i] | 스택 상태 (인덱스) | 동작                   | answer        |
| --- | ------- | ------------------ | ---------------------- | ------------- |
| 0   | 3       | []                 | push(0)                | [-1,-1,-1,-1] |
| 1   | 5       | [0]                | 3<5 → pop, answer[0]=5 | [5,-1,-1,-1]  |
| 1   | 5       | []                 | push(1)                | [5,-1,-1,-1]  |
| 2   | 2       | [1]                | 5>2 → push(2)          | [5,-1,-1,-1]  |
| 3   | 7       | [1,2]              | 2<7 → pop, answer[2]=7 | [5,-1,7,-1]   |
| 3   | 7       | [1]                | 5<7 → pop, answer[1]=7 | [5,7,7,-1]    |
| 3   | 7       | []                 | push(3)                | [5,7,7,-1]    |

스택에 남은 인덱스(3)는 오큰수가 없으므로 -1 유지.

### 3. 결과 출력

```javascript
console.log(answer.join(" "));
```

최종 결과: `5 7 7 -1`
