# 리모컨

[백준 1107번: 리모컨](https://www.acmicpc.net/problem/1107)

## 문제 파악하기

현재 채널 100에서 목표 채널 N으로 이동할 때 **최소 버튼 클릭 수**를 구하는 문제입니다.

### 사용 가능한 버튼

- 숫자 버튼 (0~9): 일부 고장났을 수 있음
- `+` 버튼: 채널 +1
- `-` 버튼: 채널 -1

> [!NOTE]
> 숫자 버튼으로 특정 채널에 도달한 후, +/- 버튼으로 미세 조정하는 방식입니다.
> 목표 채널과 가까운 채널 중 숫자로 입력 가능한 채널을 찾아야 합니다.

## 풀이

**브루트포스 (완전 탐색)** 로 해결합니다.

### 핵심 아이디어

0부터 1,000,000까지 모든 채널에 대해:

1. 숫자 버튼으로 입력 가능한지 확인
2. 가능하다면 `자릿수 + |목표 - 해당채널|` 계산
3. 최솟값 갱신

### 1. 기본값 설정

```javascript
let answer = Math.abs(target - 100);
```

+/- 버튼만으로 이동하는 경우 (숫자 버튼 사용 안 함)

### 2. 고장난 버튼 체크

```javascript
const broken = Array(10).fill(false);

for (const num of nums) {
  broken[num] = true;
}
```

### 3. 채널 입력 가능 여부 확인

```javascript
const canType = (num) => {
  if (num === 0) return !broken[0];
  while (num > 0) {
    const d = num % 10;
    if (broken[d]) return false;
    num = Math.floor(num / 10);
  }
  return true;
};
```

각 자릿수가 고장난 버튼인지 확인합니다.

> [!TIP] > `num === 0`인 경우를 따로 처리해야 합니다!
> while 문은 `num > 0`일 때만 돌기 때문에 0은 체크되지 않습니다.

### 4. 모든 채널 탐색

```javascript
const len = (num) => (num === 0 ? 1 : String(num).length);

for (let i = 0; i < 1000001; i++) {
  if (!canType(i)) continue;
  const r = len(i) + Math.abs(target - i);
  if (r < answer) answer = r;
}
console.log(answer);
```

## 동작 예시

```
목표: 5457, 고장: [6, 7, 8]

기본값: |5457 - 100| = 5357

탐색:
- 5455 입력 가능 → 4 + |5457 - 5455| = 4 + 2 = 6
- 5459 입력 가능 → 4 + |5457 - 5459| = 4 + 2 = 6

최솟값: 6
```

## 핵심 정리

| 방법       | 버튼 수                 |
| ---------- | ----------------------- |
| +/- 버튼만 | \|target - 100\|        |
| 숫자 + +/- | len(i) + \|target - i\| |

### 왜 1,000,000까지 탐색하는가?

- 목표 채널: 최대 500,000
- 예: 목표가 500,000이고 5, 0만 고장이면 499,999나 500,001에서 시작해야 함
- 안전하게 1,000,000까지 탐색

### 시간 복잡도

- 채널 탐색: O(1,000,000)
- 각 채널 입력 가능 확인: O(6) (최대 6자리)
- 총 **O(N)** (N = 1,000,000)
