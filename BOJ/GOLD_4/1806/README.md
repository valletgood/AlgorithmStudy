# 부분합

[백준 1806번: 부분합](https://www.acmicpc.net/problem/1806)

## 문제 파악하기

연속된 수들의 부분합 중 **합이 S 이상**이면서 **가장 짧은 길이**를 구하는 문제입니다.

### 조건

- 연속된 수들의 합이 S 이상
- 그 중 가장 짧은 구간의 길이
- 불가능하면 0 출력

> [!NOTE]
> 연속 구간의 합 → **투 포인터**로 효율적으로 탐색할 수 있습니다!

## 풀이

**투 포인터 (Two Pointers)** 로 해결합니다.

### 핵심 아이디어

- `l`, `r` 두 포인터로 구간 [l, r) 관리
- 합이 S 이상이면 → 왼쪽 축소 (더 짧은 구간 찾기)
- 합이 S 미만이면 → 오른쪽 확장 (합 늘리기)

### 1. 초기화

```javascript
const MAX = 100001;

let l = 0;
let r = 1;
let answer = MAX;
let sum = nums[0];
```

- 초기 구간: [0, 1) → nums[0]만 포함
- answer를 최댓값으로 초기화

### 2. 투 포인터 이동

```javascript
while (l < r && l <= N && r <= N) {
  if (sum >= S) {
    answer = Math.min(r - l, answer);
    sum -= nums[l];
    l++;
  } else {
    const right = nums[r];
    sum += right;
    r++;
  }
}
```

> [!TIP] > **투 포인터 동작**
>
> - `sum >= S`: 조건 만족! 길이 갱신 후 왼쪽 줄이기
> - `sum < S`: 합이 부족하니 오른쪽 늘리기

### 3. 결과 출력

```javascript
console.log(answer === MAX ? 0 : answer);
```

조건을 만족하는 구간이 없으면 0 출력

## 동작 예시

```
N=10, S=15
nums: [5, 1, 3, 5, 10, 7, 4, 9, 2, 8]

l=0, r=1, sum=5  < 15 → r++
l=0, r=2, sum=6  < 15 → r++
l=0, r=3, sum=9  < 15 → r++
l=0, r=4, sum=14 < 15 → r++
l=0, r=5, sum=24 >= 15 → answer=5, l++
l=1, r=5, sum=19 >= 15 → answer=4, l++
l=2, r=5, sum=18 >= 15 → answer=3, l++
l=3, r=5, sum=15 >= 15 → answer=2, l++
l=4, r=5, sum=10 < 15 → r++
l=4, r=6, sum=17 >= 15 → answer=2, l++
...

최종 answer = 2
```

## 핵심 정리

| 조건     | 동작           | 이유              |
| -------- | -------------- | ----------------- |
| sum >= S | l++, 길이 갱신 | 더 짧은 구간 찾기 |
| sum < S  | r++            | 합을 늘리기       |

### 왜 투 포인터인가?

- 모든 양수 → 오른쪽 확장하면 합 증가, 왼쪽 축소하면 합 감소
- 단조성이 보장되어 O(N)에 탐색 가능

### 시간 복잡도

- 각 포인터가 최대 N번 이동
- **O(N)**
