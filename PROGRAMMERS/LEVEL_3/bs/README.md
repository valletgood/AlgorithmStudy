## 문제

https://school.programmers.co.kr/learn/courses/30/lessons/12979

### 문제 파악하기

위 문제는 프로그래머스 기지국 설치 문제이다.

1부터 n까지 번호가 붙은 아파트가 일렬로 있고, 일부 아파트에는 기지국이 설치되어 있다. 기지국의 전파 도달 거리가 w일 때, 모든 아파트에 전파가 도달하도록 추가로 설치해야 하는 기지국의 최소 개수를 구하는 문제다.

> [!NOTE] > **핵심 아이디어**: 기존 기지국들 사이의 **빈 구간**을 찾고, 각 빈 구간을 커버하는데 필요한 최소 기지국 수를 계산한다.
>
> - 기지국 하나가 커버하는 범위: `2 * w + 1` (자기 위치 + 양쪽 w칸)
> - 빈 구간 길이를 커버 범위로 나누고 올림하면 필요한 기지국 수

## 풀이

### 1. 빈 구간 찾기

```javascript
// 첫 번째 기지국 앞 구간
start = 1;
end = stations[0] - w - 1;

// 기지국 사이 구간
start = stations[i - 1] + w + 1;
end = stations[i] - w - 1;

// 마지막 기지국 뒤 구간
start = stations[last] + w + 1;
end = n;
```

> [!TIP] > **빈 구간 계산**
>
> - `stations[i] - w`: i번째 기지국이 커버하는 가장 왼쪽 아파트
> - `stations[i] + w`: i번째 기지국이 커버하는 가장 오른쪽 아파트
> - 따라서 빈 구간은 이전 기지국의 오른쪽 끝 +1부터 다음 기지국의 왼쪽 끝 -1까지

### 2. 필요한 기지국 수 계산

```javascript
const p = (end - start + 1) / (w * 2 + 1);
const r = (end - start + 1) % (w * 2 + 1);
answer += r === 0 ? p : Math.ceil(p);
```

- `end - start + 1`: 빈 구간의 길이
- `w * 2 + 1`: 기지국 하나가 커버하는 범위
- 나머지가 0이면 딱 나눠떨어지고, 아니면 올림 처리
- 기존 Math.ceil()을 사용했으나, 시간 초과로 인해 직접 계산으로 변경

### 3. 예시

```
n = 11, stations = [4, 11], w = 1

기지국 커버 범위: 2 * 1 + 1 = 3칸

기지국 4 커버: 3, 4, 5
기지국 11 커버: 10, 11

빈 구간 1: 1 ~ 2 (길이 2) → 기지국 1개 필요
빈 구간 2: 6 ~ 9 (길이 4) → 기지국 2개 필요

결과: 3개
```
