## 문제

https://school.programmers.co.kr/learn/courses/30/lessons/12987

### 문제 파악하기

위 문제는 프로그래머스 숫자 게임 문제이다.

A팀과 B팀이 각각 숫자 카드를 내서 더 큰 숫자를 낸 팀이 승점을 얻는다. B팀이 얻을 수 있는 최대 승점을 구하는 문제다.

<aside>
💡

B팀이 승점을 최대화하려면 A팀의 숫자를 이길 수 있는 가장 작은 숫자로 이기는 것이 유리하다. 이길 수 없는 경우에는 가장 작은 숫자를 버리는 탐욕적 전략을 사용한다.

</aside>

## 풀이

### 1. 정렬

```javascript
A.sort((a, b) => a - b);
B.sort((a, b) => b - a);
```

- A팀과 B팀의 숫자를 오름차순으로 정렬한다.
- 정렬을 통해 투 포인터로 효율적인 비교가 가능해진다.

### 2. 투 포인터를 활용한 탐욕적 매칭

```javascript
let i = 0;
let j = 0;

while (i < A.length && j < B.length) {
  if (A[i] < B[j]) {
    answer++;
    i++;
    j++;
  } else {
    j++;
  }
}
```

- `i`는 A팀의 포인터, `j`는 B팀의 포인터다.
- B팀의 숫자가 A팀의 숫자보다 크면 승점을 얻고 둘 다 다음으로 이동한다.
- B팀이 이길 수 없으면 B팀의 가장 작은 숫자를 버리고(`j++`) A팀의 숫자는 유지한다.

<aside>
💡

오름차순 정렬 후, B팀의 작은 숫자로 A팀의 작은 숫자를 이기는 전략이다. B팀이 이길 수 없으면 어차피 지는 것이므로 가장 작은 숫자를 소모하는 것이 최적이다.

</aside>

### 3. 결과 반환

```javascript
return answer;
```

- 누적된 승점을 반환한다.

<aside>
💡

시간 복잡도: 정렬 O(n log n) + 탐색 O(n) = O(n log n)

</aside>
