## 문제

https://school.programmers.co.kr/learn/courses/30/lessons/42884

### 문제 파악하기

위 문제는 프로그래머스 단속 카메라 문제이다.

고속도로를 이동하는 차량들의 진입/진출 지점이 주어질 때, 모든 차량이 최소 한 번은 카메라를 만나도록 설치해야 하는 카메라의 최소 개수를 구하는 문제다.

> [!NOTE] > **그리디(탐욕법) 접근**: 카메라를 최소로 설치하려면, 한 대의 카메라가 최대한 많은 차량을 감시해야 한다.
>
> 핵심 아이디어는 **진출 지점이 빠른 차량 순서로 정렬**하는 것!
>
> - 진출 지점에 카메라를 설치하면 해당 차량은 확실히 감시됨
> - 이 카메라로 감시 가능한 다른 차량들도 함께 처리됨

## 풀이

### 1. 진출 지점 기준 정렬

```javascript
routes.sort((a, b) => a[1] - b[1]);
```

- 각 차량의 진출 지점(`end`)이 빠른 순서로 정렬
- 이렇게 하면 고속도로를 먼저 빠져나가는 차량부터 처리

### 2. 그리디 탐색

```javascript
let cams = 0;
let cctv = -Infinity;

for (const [begin, end] of routes) {
  if (begin > cctv) {
    cams++;
    cctv = end;
  }
}
```

> [!TIP] > **카메라 설치 로직**
>
> - `cctv`: 마지막으로 설치한 카메라의 위치
> - `begin > cctv`: 현재 차량의 진입 지점이 카메라 위치보다 뒤에 있음 → 감시 불가능
> - 감시 불가능하면 새 카메라를 현재 차량의 진출 지점에 설치
>
> **왜 진출 지점에 설치하나?**
>
> - 진출 지점에 설치하면 해당 차량의 전체 구간 중 가장 마지막 위치
> - 이후에 올 차량들과 겹칠 확률이 최대화됨

### 3. 예시

```
차량 구간: [[-20,-15], [-14,-5], [-18,-13], [-5,-3]]
정렬 후:   [[-20,-15], [-18,-13], [-14,-5], [-5,-3]]

1. [-20,-15]: begin(-20) > cctv(-∞)? No → cams=1, cctv=-15
2. [-18,-13]: begin(-18) > cctv(-15)? No → 기존 카메라로 감시 가능
3. [-14,-5]:  begin(-14) > cctv(-15)? Yes → cams=2, cctv=-5
4. [-5,-3]:   begin(-5) > cctv(-5)? No → 기존 카메라로 감시 가능

결과: 2대
```
